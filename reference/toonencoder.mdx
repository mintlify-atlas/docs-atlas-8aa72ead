## Declaration

```swift
public final class ToonEncoder
```

`ToonEncoder` mirrors the design of Foundation's `JSONEncoder`: configure an instance once, then call `encode(_:)` as many times as needed.

```swift
let encoder = ToonEncoder()
encoder.delimiter = .tab       // tab-separated arrays for extra efficiency
encoder.keyFolding = .safe     // fold { a: { b: 1 } } â†’ a.b: 1

let data = try encoder.encode(myValue)
let toon = String(data: data, encoding: .utf8)!
```

<Note>
`ToonEncoder` is **not thread-safe**. Create a separate instance per thread, or protect access with a lock if sharing is required.
</Note>

## Methods

### encode(\_:)

```swift
public func encode<T: Encodable>(_ value: T) throws -> Data
```

Encodes `value` and returns its UTF-8 TOON representation.

| Parameter | Type | Description |
|-----------|------|-------------|
| `value` | `T: Encodable` | The value to encode. |

**Returns:** `Data` containing the UTF-8 encoded TOON text.

**Throws:** `EncodingError` on failure. Also throws `EncodingError` if `nonFiniteFloatStrategy` is `.throw` and a non-finite `Double` or `Float` is encountered.

```swift
struct Point: Encodable {
    let x: Double
    let y: Double
}

let encoder = ToonEncoder()
let data = try encoder.encode(Point(x: 1.5, y: 2.5))
print(String(data: data, encoding: .utf8)!)
// x: 1.5
// y: 2.5
```

<Accordion title="NonFiniteFloatStrategy">

```swift
public enum NonFiniteFloatStrategy: Hashable, Sendable
```

How to handle non-finite floating-point values (`nan`, `inf`, `-inf`).

| Case | Behavior |
|------|----------|
| `.null` | Encode as the TOON `null` literal. **(Default)** |
| `.throw` | Throw an `EncodingError` immediately. |
| `.convertToString(positiveInfinity:negativeInfinity:nan:)` | Encode as configurable string literals. |

```swift

// Throw on any non-finite value
encoder.nonFiniteFloatStrategy = .throw

// Encode as strings
encoder.nonFiniteFloatStrategy = .convertToString(
    positiveInfinity: "Inf",
    negativeInfinity: "-Inf",
    nan: "NaN"
)
```

<Accordion title="Acceleration">

```swift
public enum Acceleration: Hashable, Sendable
```

Optional backend used to accelerate large string serialization workloads via Metal.

| Case | Behavior |
|------|----------|
| `.disabled` | Always use the built-in CPU implementation. **(Default)** |
| `.metal(minimumStringByteCount: Int = 16_384)` | Use Metal when available and the string is at least `minimumStringByteCount` UTF-8 bytes long. |
| `.metalForced(minimumStringByteCount: Int = 16_384)` | Force the Metal backend when available, bypassing the adaptive speed gate. Useful for benchmarking. |

```swift
// Adaptive Metal acceleration for strings >= 16 KB
encoder.acceleration = .metal()

// Custom threshold: only kick in for strings >= 64 KB
encoder.acceleration = .metal(minimumStringByteCount: 65_536)

// Force Metal regardless of speed gate (benchmarking only)
encoder.acceleration = .metalForced()
```
