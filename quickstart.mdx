This guide walks you through a complete encode/decode round-trip using `ToonEncoder` and `ToonDecoder`. By the end you will have a working example and an understanding of what TOON output looks like.

<Steps>
  <Step title="Install ToonKit">
    Follow the [Installation guide](/installation) to add ToonKit to your project, then import the module:

    ```swift
    import Toon
    ```
  </Step>

  <Step title="Define a Codable type">
    ToonKit works with any type that conforms to `Codable`. No special protocols or annotations are needed:

    ```swift
    struct User: Codable, Equatable {
        let id: Int
        let name: String
        let tags: [String]
    }
    ```
  </Step>

  <Step title="Encode to TOON">
    Create a `ToonEncoder` and call `encode(_:)`. The encoder returns `Data` containing UTF-8 TOON text:

    ```swift
    let user = User(id: 1, name: "Ada Lovelace", tags: ["coding", "maths"])

    let encoder = ToonEncoder()
    let data = try encoder.encode(user)

    // The encoded TOON text looks like:
    // id: 1
    // name: Ada Lovelace
    // tags[2]: coding,maths
    ```

    Notice how TOON encodes the `tags` array as a single inline line with a declared length prefix (`[2]`). This is what produces the token savings compared to JSON.
  </Step>

  <Step title="Decode from TOON">
    Create a `ToonDecoder` and call `decode(_:from:)` to reconstruct the original value:

    ```swift
    let decoder = ToonDecoder()
    let decoded = try decoder.decode(User.self, from: data)
    assert(decoded == user)
    ```
  </Step>

  <Step title="Inspect the output">
    Convert the encoded `Data` to a `String` to print or log the TOON representation:

    ```swift
    if let toonString = String(data: data, encoding: .utf8) {
        print(toonString)
    }
    // id: 1
    // name: Ada Lovelace
    // tags[2]: coding,maths
    ```
  </Step>
</Steps>

## Nested structs and arrays

TOON handles nested objects and tabular arrays of structs automatically. The encoder uses YAML-style indentation for nested objects and CSV-style tabular rows for uniform arrays of objects:

```swift
struct Address: Codable {
    let street: String
    let city: String
}

struct Order: Codable {
    let id: Int
    let amount: Double
    let status: String
}

struct Customer: Codable {
    let name: String
    let address: Address
    let orders: [Order]
}

let customer = Customer(
    name: "Ada Lovelace",
    address: Address(street: "1 Math Lane", city: "London"),
    orders: [
        Order(id: 1001, amount: 49.99, status: "shipped"),
        Order(id: 1002, amount: 12.50, status: "pending"),
    ]
)

let encoder = ToonEncoder()
let data = try encoder.encode(customer)

// name: Ada Lovelace
//
// address:
//   street: 1 Math Lane
//   city: London
//
// orders[2]{id,amount,status}:
//   1001,49.99,shipped
//   1002,12.50,pending
```

The tabular format (`orders[2]{id,amount,status}:`) is especially token-efficient for arrays of objects with a uniform set of keys.

<Note>
  `ToonEncoder` and `ToonDecoder` instances are **not thread-safe**. Create a separate instance per thread, or protect shared instances with a lock.
</Note>

<Tip>
  Both `ToonEncoder` and `ToonDecoder` expose a rich set of configuration options — delimiter, key folding, resource limits, and optional Metal acceleration.   See the [Encoder guide](/guides/encoding) and [Decoder guide](/guides/decoding) for the full reference.
</Tip>
