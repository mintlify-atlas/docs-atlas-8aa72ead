## Why ToonKit?

ToonKit serializes Swift `Codable` types to **TOON** (Token-Oriented Object Notation) — a compact, human-readable format designed from the ground up for LLM context windows.

<CardGroup cols={2}>
  <Card title="30–60% fewer tokens" icon="chart-line-down">
    TOON eliminates JSON's structural verbosity — quotes, braces, and brackets — replacing them with indentation, CSV-style rows, and declared array lengths.
  </Card>
  <Card title="Codable-compatible" icon="swift">
    Drop-in replacement for `JSONEncoder` / `JSONDecoder`. Any type already conforming to `Codable` works immediately.
  </Card>
  <Card title="Metal acceleration" icon="microchip">
    Optional GPU-backed string classification delivers up to **104x speedup** on large batched string encoding workloads.
  </Card>
  <Card title="Production-safe" icon="shield-check">
    Adaptive speed gating activates Metal only when it's measurably faster. Configurable resource limits guard against malicious input.
  </Card>
</CardGroup>

## Performance at a glance

| Benchmark | Without Metal | With dynamic Metal | Speedup |
|---|---:|---:|---:|
| Encode large escaped string array | 1 ops/s | 126 ops/s | **104.11x** |
| Encode large safe ASCII string array | 3 ops/s | 89 ops/s | **28.12x** |
| Encode structured payload | 15,904 ops/s | 16,765 ops/s | 1.05x |
| Decode structured payload | 14,342 ops/s | 14,365 ops/s | ~1.00x |

<Note>
  Measured on macOS arm64 using `swift run ToonBenchmark`. Dynamic mode activates Metal only when it outperforms CPU — decode performance is never regressed.
</Note>

## Quick example

```swift Swift
import Toon

struct User: Codable, Equatable {
    let id: Int
    let name: String
    let tags: [String]
}

let user = User(id: 1, name: "Ada Lovelace", tags: ["coding", "maths"])

// Encode to TOON
let encoder = ToonEncoder()
let data = try encoder.encode(user)
// id: 1
// name: Ada Lovelace
// tags[2]: coding,maths

// Decode from TOON
let decoder = ToonDecoder()
let decoded = try decoder.decode(User.self, from: data)
assert(decoded == user)
```
